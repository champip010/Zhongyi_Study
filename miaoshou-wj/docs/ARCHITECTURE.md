# 系统架构文档

## 概述

妙手问诊RAG系统采用模块化设计，基于现代AI技术栈构建，专门针对中文医学知识检索优化。

## 核心组件

### 1. MedicalRAG 主系统

**职责**: 系统协调和流程控制
**主要功能**:
- 文档索引构建
- 查询路由和分发
- 结果聚合和排序
- 系统状态管理

**关键方法**:
```python
class MedicalRAG:
    def build_index(self, data_paths, sample_limit, reset_qdrant, file_limit)
    def search(self, query, k)
    def self_rag_query(self, query)
    def multi_hop_retrieval(self, query, max_hops)
```

### 2. QdrantManager 向量数据库管理

**职责**: 向量存储和检索
**主要功能**:
- 向量数据库连接管理
- 文档增删改查
- 相似度搜索
- 批量操作优化

**关键方法**:
```python
class QdrantManager:
    def add_documents(self, docs, embeddings)
    def search_text(self, query, top_k)
    def get_all(self, batch_size)
    def reset_collection(self)
```

### 3. SimpleMedicalProcessor 数据处理

**职责**: 文档预处理和分块
**主要功能**:
- 多格式文档加载
- 智能分块策略
- 元数据提取
- 内容清洗和标准化

**关键方法**:
```python
class SimpleMedicalProcessor:
    def load_documents(self, data_dir, file_limit)
    def smart_chunk_documents(self, documents)
    def clean_medical_md_content(self, content, filename)
```

## 数据流

### 索引构建流程

```
原始文档 → 内容清洗 → 智能分块 → 向量编码 → 存储到Qdrant
    ↓           ↓         ↓         ↓         ↓
  文件加载   格式标准化   语义分块   BGE模型   批量写入
```

### 检索查询流程

```
用户查询 → 查询增强 → 向量检索 → 结果重排序 → 答案生成
    ↓         ↓         ↓         ↓         ↓
  输入处理   HyDE扩展   相似度计算   交叉编码器   LLM生成
```

## 技术栈

### 核心框架
- **Python 3.9+**: 主要开发语言
- **PyTorch**: 深度学习框架
- **Transformers**: Hugging Face模型库
- **LangChain**: RAG框架支持

### 向量数据库
- **Qdrant**: 高性能向量数据库
- **gRPC**: 高效通信协议
- **批量操作**: 优化写入性能

### 模型服务
- **BGE-large-zh**: 中文embedding模型
- **VLLM**: 大语言模型推理
- **SentenceTransformers**: 文本编码

### 数据处理
- **jieba**: 中文分词
- **Markdown**: 文档格式处理
- **正则表达式**: 内容清洗

## 性能优化

### 1. 向量检索优化
- **批量编码**: 减少GPU调用次数
- **索引优化**: 向量索引和元数据分离
- **缓存策略**: 热点查询结果缓存

### 2. 内存管理
- **GPU内存**: 动态调整batch_size
